---
title: Create Schedule
description: Create a new cron or interval-based schedule for a resource.
---

```
POST /api/v1/schedules
```

Create a new schedule.

**Required scope:** `manage:jobs`

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | Schedule display name. |
| `resourceType` | `string` | Yes | Resource type (`job`, `pipeline`, `maintenance`). |
| `resourceId` | `string` | Yes | ID of the resource to schedule. |
| `scheduleExpression` | `string` | Yes | Cron expression (e.g., `0 2 * * *`) or interval. |
| `timezone` | `string` | No | Timezone for the schedule. Defaults to `UTC`. |
| `active` | `boolean` | No | Whether the schedule is active. Defaults to `true`. |
| `startDate` | `string` | No | ISO 8601 date when the schedule becomes active. |
| `endDate` | `string` | No | ISO 8601 date when the schedule expires. |
| `skipIfRunning` | `boolean` | No | Skip execution if the previous run is still active. |
| `catchUpMissed` | `boolean` | No | Execute missed runs on recovery. |
| `parameters` | `object` | No | Key-value parameters passed to the resource on execution. |

## Response `201`

Returns the created schedule object.

## Example

```bash title="cURL"
curl -X POST /api/v1/schedules \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Nightly ETL",
    "resourceType": "job",
    "resourceId": "job-001",
    "scheduleExpression": "0 2 * * *",
    "timezone": "America/New_York",
    "skipIfRunning": true
  }'
```

```json title="Response"
{
  "id": "sched-new",
  "name": "Nightly ETL",
  "resourceType": "job",
  "resourceId": "job-001",
  "scheduleExpression": "0 2 * * *",
  "timezone": "America/New_York",
  "active": true,
  "skipIfRunning": true,
  "createdAt": "2024-03-01T09:00:00Z"
}
```
