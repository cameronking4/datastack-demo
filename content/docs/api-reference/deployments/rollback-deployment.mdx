---
title: Rollback Deployment
description: Rollback to a previous deployment version.
---

```
POST /api/v1/deployments/{deploymentId}/rollback
```

Rollback to the state of a previous deployment. Creates a new deployment that reverts changed layers.

**Required scope:** `manage:deployments`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `deploymentId` | `string` | The deployment identifier to rollback from. |

## Response `202`

| Field | Type | Description |
|---|---|---|
| `id` | `string` | New rollback deployment identifier. |
| `stackId` | `string` | Stack identifier. |
| `stackName` | `string` | Stack display name. |
| `version` | `number` | Stack version being rolled back to. |
| `environment` | `string` | Target environment. |
| `status` | `string` | `IN_PROGRESS`. |
| `rollbackFrom` | `string` | Deployment ID being rolled back. |
| `rollbackTo` | `string` | Target deployment state. |
| `steps` | `object[]` | Per-layer rollback steps. |
| `startedAt` | `string` | ISO 8601 start timestamp. |
| `startedBy` | `string` | Initiator identifier. |

## Example

```bash title="cURL"
curl -X POST /api/v1/deployments/deploy-003/rollback \
  -H "Authorization: Bearer <accessToken>"
```
