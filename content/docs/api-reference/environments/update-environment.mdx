---
title: Update Environment
description: Update environment settings, promotion rules, or access controls.
---

```
PATCH /api/v1/environments/{environmentId}
```

Update an existing environment.

**Required scope:** `manage:deployments`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `environmentId` | `string` | The environment identifier. |

## Request Body

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Display name. |
| `description` | `string` | Environment description. |
| `color` | `string` | Display color. |
| `promotionRules` | `object` | Promotion configuration. |
| `accessControl` | `object` | Access restrictions. |
| `variables` | `object` | Key-value environment variables. |
| `protected` | `boolean` | Whether the environment is protected. |

## Response `200`

Returns the updated environment object.

## Example

```bash title="cURL"
curl -X PATCH /api/v1/environments/env-001 \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "promotionRules": {
      "requiresApproval": true,
      "approvers": ["user-001", "user-002"]
    }
  }'
```
