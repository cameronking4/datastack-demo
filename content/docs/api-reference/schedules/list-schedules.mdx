---
title: List Schedules
description: List unified schedules across jobs, pipelines, and maintenance windows.
---

```
GET /api/v1/schedules
```

Returns all schedules configured in the workspace.

**Required scope:** `read:schedules`

## Query Parameters

| Parameter | Type | Description |
|---|---|---|
| `resourceType` | `string` | Filter by resource type (`job`, `pipeline`, `maintenance`). |
| `active` | `boolean` | Filter by active status. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `schedules` | `object[]` | Array of schedule objects. |
| `schedules[].id` | `string` | Schedule identifier. |
| `schedules[].name` | `string` | Display name. |
| `schedules[].resourceType` | `string` | Associated resource type. |
| `schedules[].resourceId` | `string` | Associated resource identifier. |
| `schedules[].scheduleExpression` | `string` | Cron or interval expression. |
| `schedules[].timezone` | `string` | IANA timezone. |
| `schedules[].active` | `boolean` | Whether the schedule is active. |
| `schedules[].nextFireTime` | `string` | ISO 8601 timestamp of next execution. |
| `schedules[].createdAt` | `string` | ISO 8601 creation timestamp. |
| `totalCount` | `number` | Total number of schedules. |

## Example

```bash title="cURL"
curl "/api/v1/schedules?resourceType=job&active=true" \
  -H "Authorization: Bearer <accessToken>"
```

```json title="Response"
{
  "schedules": [
    {
      "id": "sched-001",
      "name": "Daily ETL Schedule",
      "resourceType": "job",
      "resourceId": "1001",
      "scheduleExpression": "0 0 2 * * ?",
      "timezone": "America/New_York",
      "active": true,
      "nextFireTime": "2024-06-11T02:00:00Z",
      "createdAt": "2024-03-01T10:00:00Z"
    }
  ],
  "totalCount": 1
}
```
