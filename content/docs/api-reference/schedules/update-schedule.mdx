title: Update schedule
description: Update schedule expression, timezone, or activation state
---

`PATCH /api/v1/schedules/{scheduleId}`

Update schedule expression, timezone, or activation state

## Path Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `scheduleId` | `string` | Yes | The scheduleId value. |

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | No | The name value. |
| `scheduleExpression` | `string` | No | The scheduleExpression value. |
| `timezone` | `string` | No | The timezone value. |
| `active` | `boolean` | No | The active value. |
| `startDate` | `string` | No | The startDate value. |
| `endDate` | `string` | No | The endDate value. |
| `skipIfRunning` | `boolean` | No | The skipIfRunning value. |
| `catchUpMissed` | `boolean` | No | The catchUpMissed value. |
| `parameters` | `object` | No | The parameters value. |

## Response `200`

Success

## Example

```bash title="cURL"
curl -X PATCH "/api/v1/schedules/{scheduleId}" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
  "name": "example-name",
  "scheduleExpression": "example-scheduleExpression",
  "timezone": "example-timezone",
  "active": true,
  "startDate": "example-startDate",
  "endDate": "example-endDate",
  "skipIfRunning": true,
  "catchUpMissed": true,
  "parameters": {}
}'
```
