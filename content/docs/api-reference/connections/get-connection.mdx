---
title: Get Connection
description: Retrieve a connection by identifier.
---

```
GET /api/v1/connections/{connectionId}
```

Returns a single connection with full configuration details.

**Required scope:** `read:connections`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `connectionId` | `string` | Connection identifier. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Connection identifier. |
| `name` | `string` | Display name. |
| `type` | `string` | Connection type. |
| `host` | `string` | Host or endpoint URI. |
| `status` | `string` | Status (`ACTIVE`, `PENDING_VALIDATION`, `FAILED`). |
| `workspaceId` | `string` | Workspace identifier. |
| `secretScope` | `string` | Secret scope for credentials. |
| `properties` | `object` | Type-specific configuration. |
| `createdAt` | `string` | ISO 8601 creation timestamp. |
| `updatedAt` | `string` | ISO 8601 last-updated timestamp. |
| `createdBy` | `string` | Creator identity. |
| `lastTestedAt` | `string` | ISO 8601 last test timestamp. |
| `lastTestStatus` | `string` | Last test result. |

## Example

```bash title="cURL"
curl "/api/v1/connections/conn-001" \
  -H "Authorization: Bearer <accessToken>"
```
