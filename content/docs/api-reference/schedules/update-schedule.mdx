---
title: Update Schedule
description: Update schedule expression, timezone, or activation state.
---

```
PATCH /api/v1/schedules/{scheduleId}
```

Update an existing schedule.

**Required scope:** `manage:jobs`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `scheduleId` | `string` | The schedule identifier. |

## Request Body

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Display name. |
| `scheduleExpression` | `string` | Cron or interval expression. |
| `timezone` | `string` | Schedule timezone. |
| `active` | `boolean` | Whether the schedule is active. |
| `startDate` | `string` | ISO 8601 start date. |
| `endDate` | `string` | ISO 8601 end date. |
| `skipIfRunning` | `boolean` | Whether to skip if previous run is active. |
| `catchUpMissed` | `boolean` | Whether to catch up missed runs. |
| `parameters` | `object` | Key-value execution parameters. |

## Response `200`

Returns the updated schedule object.

## Example

```bash title="cURL"
curl -X PATCH /api/v1/schedules/sched-001 \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "scheduleExpression": "0 3 * * *",
    "timezone": "America/Chicago"
  }'
```
