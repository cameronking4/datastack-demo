---
title: Create Connection
description: Create a new data source or sink connection.
---

```
POST /api/v1/connections
```

Creates a new external data connection.

**Required scope:** `manage:connections`

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | Display name. |
| `type` | `string` | Yes | Connection type (`S3`, `POSTGRESQL`, `MYSQL`, `KAFKA`, `SNOWFLAKE`, `BIGQUERY`, `REDSHIFT`, `MONGODB`, `AZURE_BLOB`, `GCS`). |
| `host` | `string` | Yes | Host or endpoint URI. |
| `port` | `integer` | No | Port number. |
| `workspaceId` | `string` | Yes | Target workspace. |
| `authMethod` | `string` | Yes | Authentication method (`API_KEY`, `OAUTH2`, `SERVICE_ACCOUNT`, `IAM_ROLE`, `USERNAME_PASSWORD`). |
| `credentialRef` | `string` | No | Reference to stored credentials. |
| `healthCheck` | `object` | No | Health-check configuration. |
| `healthCheck.enabled` | `boolean` | No | Enable periodic health checks. |
| `healthCheck.intervalSeconds` | `integer` | No | Seconds between checks. |
| `properties` | `object` | No | Type-specific configuration. |
| `tags` | `object` | No | Key-value metadata tags. |

## Response `201`

Returns the created connection object with `status` set to `PENDING_VALIDATION`.

## Example

```bash title="cURL"
curl -X POST "/api/v1/connections" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Staging Postgres",
    "type": "POSTGRESQL",
    "host": "staging-db.acme.internal",
    "workspaceId": "ws-001",
    "authMethod": "USERNAME_PASSWORD",
    "credentialRef": "db-staging",
    "properties": { "database": "analytics", "sslMode": "require" }
  }'
```
