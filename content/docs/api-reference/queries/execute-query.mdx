---
title: Execute Query
description: Execute a saved SQL query against its warehouse.
---

```
POST /api/v1/queries/{queryId}/execute
```

Submit a saved SQL query for execution against its associated warehouse. The execution runs asynchronously.

**Required scope:** `manage:queries`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `queryId` | `string` | The query identifier. |

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `parameters` | `object` | No | Key-value parameter bindings for parameterized queries. |
| `limit` | `number` | No | Maximum rows to return. Defaults to `1000`. |

## Response `202`

| Field | Type | Description |
|---|---|---|
| `executionId` | `string` | Execution identifier. |
| `queryId` | `string` | Query identifier. |
| `warehouseId` | `string` | Warehouse used for execution. |
| `status` | `string` | `RUNNING`. |
| `parameters` | `object` | Parameter bindings used. |
| `limit` | `number` | Row limit. |
| `submittedAt` | `string` | ISO 8601 submission timestamp. |
| `submittedBy` | `string` | Submitter identifier. |

## Example

```bash title="cURL"
curl -X POST /api/v1/queries/query-001/execute \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{ "limit": 100 }'
```

```json title="Response"
{
  "executionId": "exec-new",
  "queryId": "query-001",
  "warehouseId": "wh-001",
  "status": "RUNNING",
  "parameters": {},
  "limit": 100,
  "submittedAt": "2024-06-10T10:00:00Z",
  "submittedBy": "api"
}
```
