---
title: Lock Stack
description: Acquire an exclusive lock on a stack to prevent concurrent modifications.
---

```
POST /api/v1/stacks/{stackId}/lock
```

Acquire an exclusive lock on a stack to prevent concurrent modifications during deployments or maintenance.

**Required scope:** `manage:stacks`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `stackId` | `string` | The stack identifier. |

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `reason` | `string` | No | Human-readable reason for locking. |
| `ttlSeconds` | `number` | No | Lock time-to-live in seconds. |

## Response `200`

Returns the lock details.

## Errors

| Code | Description |
|---|---|
| `409` | Stack is already locked. |

## Example

```bash title="cURL"
curl -X POST /api/v1/stacks/stack-001/lock \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "Production deployment in progress",
    "ttlSeconds": 1800
  }'
```

```json title="Response"
{
  "message": "Stack stack-001 locked."
}
```
