---
title: Refresh Token
description: Exchange a refresh token for a new access token without re-authenticating.
---

```
POST /api/v1/auth/token/refresh
```

Exchange a refresh token for a new access token. This avoids requiring the client to re-authenticate with credentials.

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `refreshToken` | `string` | Yes | The refresh token returned from [Generate Token](/docs/api-reference/authentication/generate-token). |
| `scopes` | `string[]` | No | Optionally narrow the scopes for the new token. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `accessToken` | `string` | New JWT access token. |
| `refreshToken` | `string` | New refresh token. |
| `tokenType` | `string` | Always `Bearer`. |
| `expiresIn` | `number` | Token lifetime in seconds. |
| `scopes` | `string[]` | Granted scopes. |

## Errors

| Code | Description |
|---|---|
| `401` | Invalid or expired refresh token. |

## Example

```bash title="cURL"
curl -X POST /api/v1/auth/token/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "drt_refresh_demo_token"
  }'
```

```json title="Response"
{
  "accessToken": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9.new-token",
  "refreshToken": "drt_refresh_new_token",
  "tokenType": "Bearer",
  "expiresIn": 3600,
  "scopes": ["read:clusters", "manage:jobs"]
}
```
