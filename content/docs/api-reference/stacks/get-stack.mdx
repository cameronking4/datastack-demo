---
title: Get Stack
description: Retrieve a stack by identifier with full module details.
---

```
GET /api/v1/stacks/{stackId}
```

Returns a single stack with detailed module graph, policies, compliance metadata, and revision history.

**Required scope:** `read:stacks`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `stackId` | `string` | Stack identifier. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Stack identifier. |
| `name` | `string` | Display name. |
| `description` | `string` | Stack description. |
| `workspaceId` | `string` | Workspace identifier. |
| `lifecycleState` | `string` | Lifecycle state (`DRAFT`, `ACTIVE`, `DEPLOYING`, `FAILED`, `DECOMMISSIONED`). |
| `environment` | `string` | Target environment (`development`, `staging`, `production`). |
| `owner` | `string` | Owner identity. |
| `modules` | `object[]` | Infrastructure modules with health and dependency info. |
| `modules[].name` | `string` | Module name. |
| `modules[].provider` | `string` | Cloud provider. |
| `modules[].resourceType` | `string` | Resource type. |
| `modules[].resourceId` | `string` | Referenced resource identifier. |
| `modules[].version` | `string` | Module version. |
| `modules[].inputs` | `object` | Module input parameters. |
| `modules[].outputs` | `string[]` | Module output names. |
| `modules[].dependsOn` | `string[]` | Module dependencies. |
| `modules[].healthCheck` | `object` | Health check configuration. |
| `policies` | `object` | Stack governance policies (drift detection, rollback, approval). |
| `compliance` | `object` | Compliance metadata (frameworks, data classification). |
| `revision` | `number` | Stack revision number. |
| `tags` | `object` | Key-value tags. |
| `createdAt` | `string` | ISO 8601 creation timestamp. |
| `updatedAt` | `string` | ISO 8601 last-updated timestamp. |
| `createdBy` | `string` | Creator identity. |

## Example

```bash title="cURL"
curl "/api/v1/stacks/stack-001" \
  -H "Authorization: Bearer <accessToken>"
```
