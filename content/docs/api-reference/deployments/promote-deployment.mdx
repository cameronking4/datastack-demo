---
title: Promote Deployment
description: Promote a deployment to the next environment.
---

```
POST /api/v1/deployments/{deploymentId}/promote
```

Promotes a deployment to the next environment (e.g., development to staging, staging to production).

**Required scope:** `manage:deployments`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `deploymentId` | `string` | Source deployment identifier. |

## Request Body

| Field | Type | Default | Description |
|---|---|---|---|
| `targetEnvironment` | `string` | Next environment | Target environment to promote to. |

## Response `202`

Returns the new deployment object with status `IN_PROGRESS`.

## Example

```bash title="cURL"
curl -X POST "/api/v1/deployments/deploy-002/promote" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{ "targetEnvironment": "PRODUCTION" }'
```

```json title="Response"
  "stackId": "stack-001",

  "promotedFrom": "deploy-002",

    { "layer": "ingestion", "resourceType": "pipeline", "resourceId": "pipe-001", "action": "CREATE", "status": "PENDING" },
    { "layer": "compute", "resourceType": "cluster", "resourceId": "cluster-001", "action": "CREATE", "status": "PENDING" }

```
