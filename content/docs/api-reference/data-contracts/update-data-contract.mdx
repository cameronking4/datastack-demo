---
title: Update Data Contract
description: Update an existing data contract.
---

```
PATCH /api/v1/data-contracts/{contractId}
```

Update fields on an existing data contract. Only the provided fields are changed; omitted fields remain unchanged. The version is automatically incremented.

**Required scope:** `manage:data-contracts`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `contractId` | `string` | Contract identifier. |

## Request Body

All fields are optional. Only included fields are updated.

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Contract display name. |
| `description` | `string` | Human-readable description. |
| `status` | `string` | Contract status (`ACTIVE`, `DRAFT`, `EXPIRED`, `ARCHIVED`). |
| `producer` | `object` | Producer team details. |
| `consumer` | `object` | Consumer team details. |
| `schema` | `object[]` | Updated column definitions. |
| `sla` | `object` | Updated SLA thresholds. |
| `qualityRules` | `object[]` | Updated quality rules. |
| `tags` | `object` | Updated key-value metadata tags. |
| `expiresAt` | `string` | ISO 8601 expiration timestamp. |

## Response `200`

Returns the updated data contract object.

## Example

```bash title="cURL"
curl -X PATCH "/api/v1/data-contracts/dc-001" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "ACTIVE",
    "sla": { "freshnessMinutes": 10, "uptimePercent": 99.95, "maxNullPercent": 0.5 }
  }'
```
