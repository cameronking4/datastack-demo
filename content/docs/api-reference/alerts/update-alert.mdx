---
title: Update Alert Rule
description: Update alert rule conditions, channels, or enabled state.
---

```
PATCH /api/v1/alerts/{alertId}
```

Updates one or more fields of an existing alert rule.

**Required scope:** `manage:alerts`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `alertId` | `string` | Alert rule identifier. |

## Request Body

All fields are optional. Only provided fields are updated.

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Alert rule name. |
| `condition` | `object` | Trigger condition (metric, operator, threshold, windowMinutes). |
| `severity` | `string` | Severity level (`info`, `warning`, `critical`). |
| `channels` | `object[]` | Notification channels (`{type, target}`). |
| `enabled` | `boolean` | Whether the alert is enabled. |
| `cooldownMinutes` | `integer` | Minimum minutes between alerts. |

## Response `200`

Returns the updated alert rule object.

## Example

```bash title="cURL"
curl -X PATCH "/api/v1/alerts/alert-001" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{ "enabled": false }'
```
