---
title: Verify Data Contract
description: Run verification checks against a data contract.
---

```
POST /api/v1/data-contracts/{contractId}/verify
```

Runs verification checks against a data contract to validate schema conformance, SLA compliance, and quality rules.

**Required scope:** `manage:data-contracts`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `contractId` | `string` | Contract identifier. |

## Request Body

The request body is optional. If omitted, all checks run with default settings.

| Field | Type | Required | Description |
|---|---|---|---|
| `checks` | `string[]` | No | Checks to run (`SCHEMA`, `SLA`, `QUALITY`, `FRESHNESS`, `ALL`). Defaults to `["ALL"]`. |
| `sampleSize` | `integer` | No | Number of rows to sample. Defaults to `10000`. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `contractId` | `string` | Contract identifier. |
| `verificationId` | `string` | Unique verification run identifier. |
| `status` | `string` | Overall result (`PASSED`, `FAILED`). |
| `checksRun` | `string[]` | List of checks that were executed. |
| `sampleSize` | `number` | Rows sampled. |
| `results.schema` | `object` | Schema check results. |
| `results.schema.status` | `string` | `PASSED` or `FAILED`. |
| `results.schema.columnsChecked` | `number` | Number of columns validated. |
| `results.schema.missingColumns` | `string[]` | Columns missing from the table. |
| `results.schema.typeMismatches` | `object[]` | Columns with type mismatches. |
| `results.sla` | `object` | SLA check results. |
| `results.sla.status` | `string` | `PASSED` or `FAILED`. |
| `results.sla.freshnessMinutes` | `number` | Actual data freshness in minutes. |
| `results.sla.freshnessThreshold` | `number` | Configured freshness threshold. |
| `results.sla.uptimePercent` | `number` | Actual uptime percentage. |
| `results.sla.uptimeThreshold` | `number` | Configured uptime threshold. |
| `results.quality` | `object` | Quality check results. |
| `results.quality.status` | `string` | `PASSED` or `FAILED`. |
| `results.quality.rulesEvaluated` | `number` | Total rules checked. |
| `results.quality.rulesPassed` | `number` | Rules that passed. |
| `results.quality.rulesFailed` | `number` | Rules that failed. |
| `results.quality.details` | `object[]` | Per-rule results. |
| `results.freshness` | `object` | Freshness check results. |
| `results.freshness.status` | `string` | `PASSED` or `FAILED`. |
| `results.freshness.lastRecordAt` | `string` | ISO 8601 timestamp of the most recent record. |
| `results.freshness.latencyMinutes` | `number` | Current latency in minutes. |
| `results.freshness.threshold` | `number` | Configured freshness threshold. |
| `startedAt` | `string` | ISO 8601 timestamp when the verification started. |
| `completedAt` | `string` | ISO 8601 timestamp when the verification completed. |

## Example

```bash title="cURL"
curl -X POST "/api/v1/data-contracts/dc-001/verify" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
    "checks": ["SCHEMA", "QUALITY"],
    "sampleSize": 5000
  }'
```
