---
title: Reset Job
description: Reset a job's state, optionally from a specific checkpoint.
---

```
POST /api/v1/jobs/{jobId}/reset
```

Queues a state reset for a job. The next run will start from the specified checkpoint or from scratch if none is provided.

**Required scope:** `manage:jobs`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `jobId` | `string` | The job identifier. |

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `fromCheckpoint` | `string` | No | Checkpoint identifier to reset from. When omitted the full state is cleared. |
| `clearState` | `boolean` | No | Whether to clear all persisted state. Defaults to `false`. |

## Response `202`

| Field | Type | Description |
|---|---|---|
| `jobId` | `string` | Job identifier. |
| `resetId` | `string` | Unique reset request identifier. |
| `fromCheckpoint` | `string\|null` | Checkpoint used, or `null` if resetting from scratch. |
| `clearState` | `boolean` | Whether state was cleared. |
| `status` | `string` | Reset status (`RESET_ACCEPTED`). |
| `message` | `string` | Human-readable status message. |
| `requestId` | `string` | Request trace identifier. |

## Example

```bash title="cURL"
curl -X POST /api/v1/jobs/job-001/reset \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{ "fromCheckpoint": "cp-20240601", "clearState": false }'
```

```json title="Response — 202 Accepted"
{
  "jobId": "job-001",
  "resetId": "reset-1717200000000",
  "fromCheckpoint": "cp-20240601",
  "clearState": false,
  "status": "RESET_ACCEPTED",
  "message": "Job state reset has been queued. Next run will start from the specified checkpoint.",
  "requestId": "req-abc123"
}
```
