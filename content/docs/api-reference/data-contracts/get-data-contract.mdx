---
title: Get Data Contract
description: Retrieve details of a specific data contract.
---

```
GET /api/v1/data-contracts/{contractId}
```

Returns full details of a single data contract including schema, SLA, quality rules, and verification status.

**Required scope:** `read:data-contracts`

## Path Parameters

| Parameter | Type | Description |
|---|---|---|
| `contractId` | `string` | Contract identifier. |

## Response `200`

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Contract identifier. |
| `name` | `string` | Contract display name. |
| `description` | `string` | Human-readable description. |
| `producer` | `object` | Producer team details. |
| `consumer` | `object` | Consumer team details. |
| `table` | `string` | Fully-qualified table name. |
| `status` | `string` | Contract status (`ACTIVE`, `DRAFT`, `VIOLATED`, `EXPIRED`, `ARCHIVED`). |
| `version` | `string` | Semantic version. |
| `schema` | `object[]` | Column definitions with PII classification. |
| `sla` | `object` | SLA thresholds (freshness, uptime, null percentage). |
| `qualityRules` | `object[]` | Quality rule definitions. |
| `tags` | `object` | Key-value metadata tags. |
| `lastVerifiedAt` | `string\|null` | ISO 8601 timestamp of last verification. |
| `lastVerificationStatus` | `string\|null` | Last verification result (`PASSED`, `FAILED`). |
| `violationCount` | `number` | Number of active violations. |
| `expiresAt` | `string\|null` | ISO 8601 expiration timestamp. |
| `createdAt` | `string` | ISO 8601 creation timestamp. |
| `updatedAt` | `string` | ISO 8601 last-update timestamp. |
| `createdBy` | `string` | Creator identifier. |

## Example

```bash title="cURL"
curl "/api/v1/data-contracts/dc-001" \
  -H "Authorization: Bearer <accessToken>"
```
