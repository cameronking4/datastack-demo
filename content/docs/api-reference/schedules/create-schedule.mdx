---

`POST /api/v1/schedules`

Create a new cron or interval-based schedule for a resource

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | `string` | Yes | The name value. |
| `resourceType` | `string` | Yes | The resourceType value. |
| `resourceId` | `string` | Yes | The resourceId value. |
| `scheduleExpression` | `string` | Yes | The scheduleExpression value. |
| `timezone` | `string` | No | The timezone value. |
| `active` | `boolean` | No | The active value. |
| `startDate` | `string` | No | The startDate value. |
| `endDate` | `string` | No | The endDate value. |
| `skipIfRunning` | `boolean` | No | The skipIfRunning value. |
| `catchUpMissed` | `boolean` | No | The catchUpMissed value. |
| `parameters` | `object` | No | The parameters value. |

## Response `201`

Created

## Example

```bash title="cURL"
curl -X POST "/api/v1/schedules" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
  "name": "example-name",
  "resourceType": "example-resourceType",
  "resourceId": "example-resourceId",
  "scheduleExpression": "example-scheduleExpression",
  "timezone": "example-timezone",
  "active": true,
  "startDate": "example-startDate",
  "endDate": "example-endDate",
  "skipIfRunning": true,
  "catchUpMissed": true,
  "parameters": {}
}'
```
