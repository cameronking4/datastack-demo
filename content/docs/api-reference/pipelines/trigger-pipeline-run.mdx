`POST /api/v1/pipelines/{pipelineId}/runs`

Manually trigger a pipeline execution with optional parameter overrides

## Path Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `pipelineId` | `string` | Yes | The pipelineId value. |

## Request Body

| Field | Type | Required | Description |
|---|---|---|---|
| `parameters` | `object` | No | The parameters value. |
| `priority` | `string` | No | The priority value. |
| `dryRun` | `boolean` | No | The dryRun value. |

## Response `201`

Created

## Example

```bash title="cURL"
curl -X POST "/api/v1/pipelines/{pipelineId}/runs" \
  -H "Authorization: Bearer <accessToken>" \
  -H "Content-Type: application/json" \
  -d '{
  "parameters": {},
  "priority": "example-priority",
  "dryRun": true
}'
```
